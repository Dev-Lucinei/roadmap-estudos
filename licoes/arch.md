## 🎯 Resumo Executivo
Arquitetura de Software trata das decisões fundamentais sobre a estrutura de um sistema e as restrições que orientam seu desenvolvimento. Uma boa arquitetura foca em atributos de qualidade como escalabilidade, manutenibilidade e resiliência.

## 📚 Conteúdo Detalhado

### 1. Princípios SOLID
- **Conceito Chave:** Cinco princípios de design orientado a objetos que visam tornar o software mais compreensível e flexível.
- **Destaque:** `Single Responsibility Principle` (SRP) - uma classe ou módulo deve ter apenas uma razão para mudar.

### 2. Padrões de Projeto (Design Patterns)
- **Definição:** Soluções reutilizáveis para problemas comuns no design de software.
- **Categorias:**
  - `Criacionais`: Como os objetos são instanciados (ex: Singleton, Factory).
  - `Estruturais`: Como as classes se compõem (ex: Adapter, Decorator).
  - `Comportamentais`: Como os objetos se comunicam (ex: Observer, Strategy).

### 3. Estilos Arquiteturais
- **Monólito:** Aplicação única onde todos os componentes estão acoplados.
- **Microserviços:** Sistema dividido em pequenos serviços independentes que se comunicam via rede.

## 💡 Insights e Conexões
- **Por que importa:** Decisões arquiteturais erradas no início do projeto são as mais difíceis e caras de serem corrigidas no futuro ("Débito Técnico").
- **Conexões:** Utiliza intensamente os conceitos de **Engenharia de Software** para garantir a integridade do sistema durante o ciclo de vida.
- **Limitações:** "Over-engineering" (criar uma arquitetura mais complexa do que o necessário) é um erro comum que atrasa a entrega de valor.
