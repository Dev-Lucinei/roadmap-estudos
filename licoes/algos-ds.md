## 🎯 Resumo Executivo
Algoritmos são sequências lógicas de passos para resolver problemas, enquanto Estruturas de Dados são formas organizadas de armazenar e manipular informações. O domínio de ambos é essencial para escrever código que não apenas funciona, mas é performático em escala.

## 📚 Conteúdo Detalhado

### 1. Complexidade Big O
- **Conceito Chave:** Uma notação matemática que descreve o comportamento de um algoritmo à medida que os dados de entrada (`n`) crescem.
- **Métricas:**
  - `O(1)`: Tempo constante (ex: acessar elemento em um array por índice).
  - `O(log n)`: Tempo logarítmico (ex: busca binária).
  - `O(n)`: Tempo linear (ex: percorrer uma lista uma vez).
  - `O(n²)`: Tempo quadrático (ex: dois loops aninhados - evite!).

### 2. Estruturas Lineares
- **Arrays:** Acesso rápido por índice, mas inserção lenta no meio da lista.
- **Linked Lists:** Inserção e remoção rápidas, mas acesso sequencial lento.
- **Stacks (Pilhas):** Lógica `LIFO` (Last-In, First-Out) - o último a entrar é o primeiro a sair.

## 💡 Insights e Conexões
- **Por que importa:** Em aplicações Fullstack, um algoritmo ineficiente no backend pode causar latência excessiva e custos de servidor desnecessários.
- **Conexões:** Fundamental para entender como **Bancos de Dados** (especialmente Índices) funcionam internamente.
- **Limitações:** O foco excessivo em micro-otimização algorítmica pode, às vezes, prejudicar a legibilidade do código se não houver um problema de performance real.
